定时触发配置方式头脑风暴

旅程级定时触发，哪种配置方式对用户最直观？

A

方案 A — 自然语言句式（推荐）

一句话描述触发规则，控件嵌入句子中：

在活动有效期内，用户 的 🕐 09:00 进入旅程
选「每周」后额外显示「周几」选择；选「每月」额外显示「第几日」；自动生成 Cron 表达式，无需用户了解 Cron 语法

优点

  • 最接近用户自然表达
  • 零学习成本
  • 参考截图中的产品就是这种

缺点

  • 灵活度有限（工作日/多时间点等复杂场景需要"自定义"兜底）
B

方案 B — 分步表单（频率 + 时间 + 范围）

执行频率
每天
每周
每月
自定义
执行时间 🕐 09:00
胶囊单选频率 + 时间选择；清晰分组，适合频率和时间的独立配置

优点

  • 层次清晰，每行各司其职
  • 扩展性好（可加"时区"/"重试"等）

缺点

  • 比 A 多一行，稍显"表单感"
C

方案 C — 方案 A + 高级模式兜底

用户 🕐 09:00 进入旅程
▶ 高级模式（Cron 表达式）
0 9 * * * ← 自动生成，可手动编辑

优点

  • 简单场景走自然语言，复杂场景展开 Cron
  • 两类用户都照顾到

缺点

  • 实现略复杂（需要双向同步 UI ↔ Cron 表达式）
