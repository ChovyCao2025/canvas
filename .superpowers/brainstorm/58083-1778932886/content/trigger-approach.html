优化点 8：定时触发的定位

SCHEDULED_TRIGGER 应该保留为节点，还是升级为旅程级属性？

A

方案 A — 旅程属性（推荐）

在旅程设置里选择触发方式：实时触发 或 定时触发。定时触发时展示 Cron 配置。SCHEDULED_TRIGGER 节点从画布节点列表中移除。

优点

  • 语义更准确（旅程的触发方式）
  • 减少画布节点数量
  • 配置位置更自然

缺点

  • 需要改造旅程创建/编辑入口
  • 已有含此节点的旧画布需要迁移
B

方案 B — 保留节点 + 重设计 UI

SCHEDULED_TRIGGER 保持为画布节点，但重新设计其配置面板：参照优化点附图，用可视化 Cron 选择器替换当前简陋的文本输入。

优点

  • 改动小，不影响已有画布
  • 和其他触发节点保持一致

缺点

  • 定时逻辑本质上仍混入节点配置
  • 用户仍需"找节点"才能配置触发
C

方案 C — 两者兼有（渐进迁移）

旅程属性里新增触发方式选择，同时暂时保留 SCHEDULED_TRIGGER 节点，待稳定后再下线旧节点。

优点

  • 平滑过渡，不破坏存量数据

缺点

  • 短期维护两套逻辑
  • 用户可能感到困惑
