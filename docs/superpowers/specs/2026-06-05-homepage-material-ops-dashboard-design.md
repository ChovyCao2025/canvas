# HOME-MATERIAL-OPS: 首页运营驾驶舱轻量 Material 改版设计

## 目标

把现有首页从“通用指标概览”升级为“运营巡检工作台”。用户进入首页后，应先判断是否有风险，再查看核心指标是否稳定，最后快速进入异常处理、旅程统计或常用管理入口。

本设计采用“运营巡检型信息架构 + 轻量 Material 视觉语言”：
- 信息架构沿用运营优先：风险摘要、KPI、健康趋势、异常队列、Top 旅程和常用动作。
- 视觉语言借鉴 Material Design 3 的浅色 surface、轻边框、低饱和状态色、pill/chip 状态和圆形主按钮。
- 技术实现仍基于现有 Ant Design 与 Recharts，不引入新的 UI 框架。

## 背景

当前 `/home` 已具备完整模块：顶部标题、5 个 KPI、趋势图、快捷入口、Top 旅程、需要关注列表。问题不是缺少信息，而是首屏优先级偏“看指标”和“找入口”，异常处置被放在较后位置，彩色 KPI 卡片也让页面更像通用概览页。

改版目标是在不改变后端接口的前提下先提升首页判断效率和现代感。

## 范围

包含：
- 调整 `frontend/src/pages/home/index.tsx` 的首屏模块顺序。
- 复用 `frontend/src/pages/home/homeOverview.ts` 的现有数据结构。
- 将 KPI、异常项、Top 旅程和快捷入口改为更轻量的卡片与 chip 展示。
- 保留现有侧栏和整体应用布局，不做全站导航改版。
- 保留现有时间范围：今日、近 7 天、近 30 天。

不包含：
- 不引入 Material UI 或其它组件库。
- 不修改后端 `GET /canvas/home/overview` 接口。
- 不实现真实环比数据；环比 chip 可以先用现有语义文案或隐藏。
- 不重做画布列表、编辑器、统计页等非首页页面。

## 信息架构

首页首屏顺序：

1. 页头操作区
   - 标题：运营驾驶舱。
   - 副文案：当前时间窗口内的风险摘要。
   - 搜索入口：第一版只过滤首页已有的异常队列和 Top 旅程，提示文案为“搜索当前首页旅程”。
   - 时间范围切换。
   - 主按钮：新建画布。

2. 风险摘要条
   - 展示最高优先级异常，例如“沉睡用户召回失败率偏高”。
   - 同行展示失败执行、成功率、待处理数量。
   - 提供“查看异常”入口。
   - 无异常时展示健康状态，使用绿色低饱和提示。

3. KPI 区
   - 已发布旅程。
   - 触达用户数。
   - 执行成功率。
   - 触发次数。
   - 执行失败。
   - 卡片使用白色 surface，不再使用大面积彩色背景。

4. 执行健康趋势 + 异常队列
   - 左侧趋势图展示触发、失败；如果当前数据结构不支持成功率趋势，先不展示成功率线。
   - 右侧异常队列替代原“需要关注”后置区域，按严重程度排序。
   - 异常项提供处理、查看、编辑等语义动作。

5. Top 旅程 + 常用动作
   - Top 旅程从普通表格调整为更可扫的排行列表。
   - 常用动作从首屏右侧降级到底部，减少对异常队列的干扰。

## 视觉设计

整体原则：
- 页面背景使用接近 `#f8fafd` 的浅灰蓝。
- 内容卡片使用白色 surface、`1px` 轻边框和轻阴影。
- 卡片圆角保持 8px，和现有后台风格一致。
- 主色使用 Google blue 倾向的蓝色；风险色使用低饱和红底和深红文本。
- 状态使用 chip/pill，而不是大面积色块。

组件风格：
- 主按钮：圆形胶囊，蓝底白字。
- 次按钮：白底、轻边框、圆形胶囊。
- KPI：白底卡片 + 小圆图标 + 数字 + 状态 chip。
- 异常队列：头像/圆形标识 + 标题 + 描述 + 状态 chip。
- Top 旅程：排行圆标 + 旅程名称 + 触发量 + 成功率 chip + 失败数。
- 图表：保留 Recharts，线条颜色收敛为蓝色和红色，网格线降低对比度。

## 数据与行为

第一版只消费现有 `HomeOverview`：
- `summary` 用于风险摘要和 KPI。
- `trend` 用于执行健康趋势。
- `topCanvases` 用于 Top 旅程排行。
- `attentionItems` 用于风险摘要和异常队列。

页头搜索：
- 输入关键字后，在前端本地过滤 `topCanvases.name` 和 `attentionItems.name`。
- 搜索不请求后端，不搜索用户、事件或全量画布。
- 清空关键字后恢复当前时间窗口的完整概览数据。

风险摘要规则：
- 如果存在 `severity === "error"` 的关注项，展示第一条 error。
- 否则如果存在 `severity === "warning"` 的关注项，展示第一条 warning。
- 否则展示“当前暂无高优先级异常”健康状态。

异常队列排序：
- error 在前。
- warning 次之。
- info 最后。
- 同级保持后端返回顺序。

动作跳转：
- `NO_RECENT_EXECUTIONS` 跳转画布编辑页。
- 其它关注项跳转画布统计页。
- 新建画布第一版可继续跳转 `/canvas`；如果列表页已有创建弹窗路由能力，再改为直接打开创建流程。

## 空态与错误态

加载态：
- 首次加载保留居中 loading。
- 已有数据刷新时不清空页面，只在按钮或局部展示 loading。

空态：
- `attentionItems` 为空时，风险摘要条展示健康状态。
- `trend` 为空时，趋势卡片展示简洁空态。
- `topCanvases` 为空时，Top 旅程展示“当前范围暂无触发数据”。

错误态：
- 聚合接口失败时保留现有错误 Alert 和重试按钮。
- 如果已有旧数据，错误 Alert 不替代旧数据。

## 可访问性与响应式

可访问性：
- 主内容标题保持清晰层级。
- chip 不只依赖颜色表达状态，需要保留文本。
- 主按钮、刷新、查看异常、查看全部等动作保留可访问名称。

响应式：
- 宽屏：趋势和异常队列并排，Top 旅程和常用动作并排。
- 中等宽度：KPI 自动换行，趋势和异常队列上下排列。
- 小屏：页头操作区纵向堆叠，卡片单列展示。

## 测试

前端测试重点：
- `buildKpiCards` 或新增展示模型函数能生成正确 KPI 顺序和文案。
- 异常队列排序函数按 error、warning、info 排序。
- 首页能渲染风险摘要、KPI、趋势、异常队列、Top 旅程和常用动作。
- 无关注项时展示健康摘要。
- 接口失败时展示错误 Alert 和重试入口。

不要求视觉快照测试；实现后通过浏览器人工检查桌面宽度下的首页首屏布局。

## 验收标准

- 首屏第一眼能看到当前是否有待处理异常。
- KPI 区视觉更克制，不再由五个大面积彩色卡片主导页面。
- “需要关注”升级为异常队列，并位于首屏右侧。
- 快捷入口降级为常用动作，不再占据趋势图右侧核心位置。
- 页面风格更接近轻量 Material，但不会脱离现有 Ant Design 后台体系。
- 不修改后端接口即可完成第一版。
