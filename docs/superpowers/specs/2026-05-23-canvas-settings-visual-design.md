# 画布设置弹窗视觉优化设计

## 背景

当前画布编辑页的“画布设置”弹窗已经承载两类配置：

1. 触发方式（实时触发 / 定时触发）
2. 执行限制（有效期、总量上限、用户上限、冷却期）

功能已基本齐备，但界面表现仍停留在“默认表单堆叠”阶段，主要问题是：

- 信息层级过平，标题、分组、控件都挤在同一视觉平面
- 触发方式是主任务，但与低频限制项混排，注意力被稀释
- 执行限制属于高级配置，却默认完全展开，增加阅读负担
- `CronBuilder` 与触发方式的关系不够紧密，切换时视觉断裂明显

这次优化只解决弹窗的视觉层级、分区结构和低风险交互，不改变后端语义和保存逻辑。

## 目标

- 让用户打开弹窗后先理解“当前触发模式是什么”
- 将高频配置和低频配置清晰分层
- 在不增加学习成本的前提下，让弹窗更像成熟产品设置面板
- 保持后台工具的克制感，不做营销式或装饰性视觉设计

## 非目标

- 不改动画布设置字段语义
- 不新增后端接口字段
- 不引入新的复杂配置流程
- 不做移动端专门适配，本次仅保证桌面端编辑页体验

## 方案选择

讨论过的方向有三类：

1. 轻量精修：仅调整间距和默认样式
2. 卡片分区：将“触发方式”和“执行限制”拆成两个有层级的设置区
3. 更强设计感：在卡片分区基础上加强状态摘要、标题层级和展开结构

最终采用 **方案 3 的视觉强度 + 方案 2 的后台克制边界**：

- 保留后台产品的稳定、清爽基调
- 使用卡片分区建立层级
- 保留轻量状态摘要，但不做过强的展示型信息
- 将执行限制收敛为可折叠高级设置区

## 结构设计

### Modal 结构

- Modal 标题保持 `画布设置`
- Modal 宽度从当前约 `480px` 提升到 `560px - 600px` 区间，默认建议 `560px`
- 弹窗主体改为两个纵向主分区：
  1. `触发方式`
  2. `执行限制`

弹窗整体仍使用干净白底，层级主要依靠以下元素建立：

- 分区卡片边框
- 统一圆角（建议 10px - 12px）
- 更明确的内边距
- 标题与说明文案的字号层次
- 轻量状态标签

不使用厚重阴影、渐变背景、大面积强调色。

### 分区卡片

两个分区均采用统一卡片结构：

- 顶部：标题 + 一句说明 + 右上角状态摘要
- 内容区：对应字段或交互控件

卡片视觉规则：

- 白底
- 浅边框
- 轻微圆角
- 使用 14px - 16px 内边距
- 卡片之间使用 12px - 16px 垂直间距

## 触发方式分区

### 视觉层级

“触发方式”是弹窗的主任务，应始终优先于执行限制被感知。

分区顶部包含：

- 标题：`触发方式`
- 辅助说明：解释实时触发与定时触发的适用场景
- 状态摘要：显示 `当前为实时` 或 `当前为定时`

状态摘要采用轻量标签样式：

- 小尺寸
- 低侵入
- 保持可扫读，不抢主标题

### 选择方式

现有裸 `Radio.Group` 改为“两块并排选择卡”：

- 左侧：`实时触发`
- 右侧：`定时触发`

每块选择卡包含两层内容：

- 主标题：选项名
- 副文案：一句话用途说明

选中态规则：

- 浅蓝底
- 蓝色边框
- 更明显的单选圆点
- 标题字重略高

未选中态规则：

- 白底
- 浅灰边框
- 无背景强调

### 定时触发展开区

当选择 `SCHEDULED` 时，`CronBuilder` 不再像独立表单项直接接在 `Radio.Group` 下方，而是在同一分区卡片内作为附加配置块出现。

附加配置块规则：

- 与选择卡保持 `12px - 16px` 的上间距
- 使用更浅的背景或内嵌边框区分
- 视觉上明确属于“定时触发”的附属设置

这样可以形成稳定的阅读顺序：

1. 选择触发模式
2. 若为定时触发，再补充时间表达式

## 执行限制分区

### 折叠策略

“执行限制”属于低频高级配置，默认不应与主配置同等展开。

因此该分区采用 **可折叠高级设置区**：

- 默认折叠
- 标题栏始终可见
- 点击标题行或右侧展开按钮可展开内容

默认展开规则：

- 若所有限制字段都为空，则默认折叠
- 若任一限制字段已有值，则默认展开

这样可以避免“用户已配置过限制，但再次打开时看不到配置内容”的问题。

### 分区头部

头部包含：

- 标题：`执行限制`
- 辅助说明：明确“留空即不限制”
- 状态摘要：
  - 无配置时：`未设置限制`
  - 有配置时：`已配置 N 项`

这里的 `N` 指以下字段中非空项数量：

- `validRange`
- `maxTotalExecutions`
- `perUserDailyLimit`
- `perUserTotalLimit`
- `cooldownSeconds`

不在标题区直接拼接完整明细，避免摘要过长。

### 展开后的提示

展开内容顶部可增加一条轻提示：

`仅在需要控频、限流或限定活动窗口时填写`

提示规则：

- 弱提示样式
- 不使用警告色
- 不带图标或只用极轻量提示图标

目的只是帮助用户理解这组字段的使用时机，而不是制造“必须阅读”的阻塞。

### 字段布局

展开后表单布局采用 `1 + 2 + 2`：

1. 第一行：`有效期`
2. 第二行：`总执行次数上限`、`用户每日上限`
3. 第三行：`用户总上限`、`冷却秒数`

原因：

- 有效期的 `RangePicker` 宽度需求最大，适合独占一行
- 数值字段两列并排能明显减少垂直长度
- 比连续五行堆叠更利于扫读

### 控件一致性

日期和数字控件需要统一表单节奏：

- 输入高度统一
- 圆角统一
- 标签和输入框间距统一
- 占位文本风格统一

避免 `RangePicker` 与 `InputNumber` 像来自不同组件系统。

## 交互规则

### 保存与关闭

- 底部操作仍保持 `保存` / `取消`
- 不新增其他操作按钮
- `保存` 仍为主按钮

### 折叠与展开

- 折叠区有短促过渡，但整体保持工具型节奏
- 不使用明显动画，不制造卡顿感
- 点击标题区与右侧箭头都可以展开或收起

### 错误与校验

- 当触发方式为 `SCHEDULED` 且未填写 cron 时，错误提示只出现在 `CronBuilder` 所在附加配置块
- 不在整个弹窗顶层显示泛化错误

### 摘要展示

摘要只承担“快速感知当前状态”的作用：

- `触发方式` 摘要显示当前模式
- `执行限制` 摘要显示是否配置及配置数量

摘要不承担详细预览职责，避免表头区域过于嘈杂。

## 实现边界

实现上只调整该 Modal 的局部结构与样式，不扩散到全站：

- 优先复用现有 `Ant Design` 组件
- 允许使用 `Collapse`，也允许用局部状态自定义可折叠卡片
- 局部样式优先挂在 `canvas-editor` 页面附近
- 不引入全局样式覆盖

推荐组件组合：

- `Modal`
- `Form`
- `Radio`
- `DatePicker.RangePicker`
- `InputNumber`
- `Tag`
- `Collapse` 或等价局部折叠实现

## 验收标准

满足以下条件视为本次视觉优化完成：

1. 打开弹窗时，用户首先感知到“触发方式”而不是长表单
2. “触发方式”和“执行限制”形成两个清晰的设置分区
3. `执行限制` 在未配置时默认折叠，在已配置时默认展开
4. `实时触发 / 定时触发` 的选中态明显优于默认单选按钮
5. `CronBuilder` 与“定时触发”之间的从属关系在视觉上明确
6. 摘要信息存在，但不会压过标题与主配置项
7. 桌面端弹窗在常见分辨率下无拥挤、错位或文字截断

## 风险与约束

- `InputNumber` 与 `RangePicker` 的统一视觉需要局部样式配合，不能只依赖默认 Ant Design 外观
- 若当前页面没有独立样式文件，可能需要新增页面级样式文件或局部类名
- 若选择 `Collapse` 默认样式，可能仍需覆盖边框、padding 和 header 高度，否则容易回到默认后台控件观感

## 结论

本次优化采用“主配置直达，高级配置折叠，状态摘要轻提示”的设计原则：

- 主路径更干净
- 高级能力仍可达
- 已配置状态可见
- 视觉层级比现状明显提升，但不会脱离后台工具语境

这是一种低风险、易实现、同时能显著改善观感的优化方式。
