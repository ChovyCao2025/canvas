# HOME-OPS: 首页运营驾驶舱设计

## 目标

把当前占位首页建设为运营视角的全局驾驶舱。首页默认展示最近 7 天数据，帮助运营快速判断活动旅程是否正常、哪些旅程效果好、哪些旅程需要介入。

## 范围

- 新增全局首页聚合接口：`GET /canvas/home/overview?days=7`。
- 替换 `frontend/src/pages/home` 占位页为彩色、可扫描的运营首页。
- 默认时间范围为最近 7 天，支持今日、最近 7 天、最近 30 天切换。
- 测试只覆盖最关键的聚合口径和首页渲染路径。

## 首页数据

核心 KPI：
- 已发布旅程数。
- 触发次数。
- 触达用户数。
- 执行成功率。
- 失败次数。

趋势：
- 按天展示触发次数和失败次数。

重点旅程：
- Top 5 已发布旅程，按触发次数降序。
- 显示旅程名称、触发次数、触达用户数、成功率、失败次数。
- 点击进入对应旅程效果看板。

需要关注：
- 近 7 天失败率较高的旅程。
- 近 7 天有失败的旅程。
- 已发布但近 7 天无触发的旅程。
- 每条提供进入效果看板或旅程编辑页的入口。

快捷入口：
- 新建画布。
- 旅程管理。
- 人群管理。
- 事件、MQ、API 配置入口。

## 接口设计

`GET /canvas/home/overview`

参数：
- `days`: `1 | 7 | 30`，默认 `7`，非法值回退为 `7`。

返回：

```json
{
  "range": { "days": 7, "since": "2026-05-17", "until": "2026-05-23" },
  "summary": {
    "publishedCanvasCount": 8,
    "totalExecutions": 1200,
    "uniqueUsers": 860,
    "successRate": "97.8%",
    "failedExecutions": 26
  },
  "trend": [

  ],
  "topCanvases": [
    {
      "canvasId": 1,
      "name": "新用户旅程",
      "total": 300,
      "uniqueUsers": 240,

}
```

## UI 设计

页面采用运营驾驶舱布局，不做营销落地页。视觉上使用浅色背景、彩色 KPI 卡、清晰图标和弱阴影，让首页比当前占位页更有完成度。

结构：
- 顶部标题、更新时间、时间范围切换。
- 第一行 5 个彩色 KPI 卡。
- 第二行左侧趋势图，右侧快捷入口。
- 第三行左侧 Top 旅程表，右侧需要关注列表。

空态：
- 没有执行数据时，KPI 显示 0，趋势显示空态。
- 没有异常时，需要关注区域显示“暂无需要关注的旅程”。

错误态：
- 聚合接口失败时展示 Ant Design `Result` 或 `Alert`，提供重试按钮。

## 数据流

前端首页根据选中的 `days` 请求 `GET /canvas/home/overview`。后端在一个接口内聚合 `canvas` 和 `canvas_execution` 数据，避免前端对每个画布发起 N+1 统计请求。

## 测试

后端保留最关键单测：
- 聚合接口在有成功、失败、不同用户时计算 KPI 正确。
- 无执行数据时返回 0 值和空趋势，不报错。

前端保留最关键测试：
- 首页能渲染 KPI、趋势区域、Top 旅程、需要关注区域。
- 接口失败时展示错误态和重试入口。

## 非目标

- 不建设管理/运维首页。
- 不展示底层异步任务、通知中心、重放队列。
- 不做复杂权限定制，不同角色先共用同一运营首页。
