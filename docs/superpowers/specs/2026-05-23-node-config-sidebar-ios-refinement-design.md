# 节点配置侧边栏 iOS 风格强化设计

## 背景

上一轮节点配置侧边栏视觉升级已经完成基础的 inspector 化改造，但用户反馈仍然集中在两个问题：

- 顶部识别区仍然不够鲜明，像 `DIRECT_CALL`、`已配置` 这类状态信息没有形成足够的视觉主次
- 表单控件仍偏默认 Ant Design 观感，下拉框和输入框看起来“原始”，缺少统一的高级感

本轮工作是在上一版 inspector 结构上继续细化视觉表现，目标是让侧边栏更接近“头部强识别 + 重 iOS 控件”的方向。

## 目标

- 将顶部识别区强化为更明确的视觉焦点
- 保留后台工具的克制感，不使用花哨渐变或过度装饰
- 让输入框和下拉框明显脱离默认 Ant Design 质感，形成更统一的 iOS 风格
- 保持现有功能和交互完全不变，仅做 UI 细化

## 非目标

- 不修改任何字段语义、数据流或保存逻辑
- 不改侧边栏分区结构
- 不新增原生移动端式滚轮 picker 或偏离桌面 Web 使用习惯的控件
- 不改左侧节点库和画布节点本体

## 最终视觉方向

采用已确认的 `B` 方向：

- 浅雾蓝头部
- 去掉右侧突兀的 `T` 标识块
- 保留 `Tagger` 胶囊、标题、双标签、说明文案和右上角状态标签
- 控件整体采用更厚、更圆、更柔和的 iOS 风格

这次不采用高饱和蓝色渐变头部，因为用户明确认为其“花里花哨”。

## 头部设计

### 整体风格

头部使用浅雾蓝底色，而不是强渐变或深重横幅：

- 背景：低饱和蓝灰色块
- 文本：深蓝灰文字
- 边界：细浅边框分隔

目标是同时满足：

- 有明显识别
- 不显得浮夸
- 与下方浅色 iOS 控件区自然衔接

### 元素构成

头部保留以下信息：

1. `Tagger` 胶囊标签
2. 节点标题
3. 双标签信息
4. 弱说明文案
5. 右上角 `已配置` 状态标签

头部明确移除：

- 右侧单独的 `T` 方块识别元素

原因是它在当前窄侧栏中会制造多余的第二视觉中心，破坏标题区的一致性。

### 颜色策略

- `Tagger` 胶囊：蓝灰边框 + 浅底
- 状态标签：与胶囊同一色系，但权重略低
- 标题：深色高字重
- 说明文案：更淡、更小

整体不能出现高亮荧光蓝、强对比玻璃球或营销式炫光。

## 控件设计

### 目标气质

控件采用“桌面 Web 中的 iOS 风格”，不是原生移动端控件复刻。

关键词：

- 更圆
- 更厚
- 更柔和
- 更统一

具体表现为：

- 输入框高度提升
- 圆角明显增加
- 边框更浅
- 内阴影和轻微高光让控件更精致
- 下拉箭头更轻、更克制

### 输入框 / 下拉框

输入和下拉统一改成以下方向：

- 高度约 `50px - 52px`
- 圆角约 `18px`
- 背景使用极浅的白到浅灰渐层
- 边框使用柔和蓝灰或浅灰
- 允许极轻的内高光和投影，营造细腻感

不使用：

- 强阴影浮空感
- 玻璃拟物过重效果
- 非桌面友好的滚轮式选择器

### 标签与字段标题

字段标题继续弱化：

- 小字号
- 低饱和灰蓝
- 与控件间保留足够留白

重点应该落在控件值本身，而不是字段 label 抢眼。

## 分支结果区

分支结果卡延续当前蓝 / 红语义，但进一步统一到新的圆角与阴影体系：

- 更圆的边角
- 更细的描边
- 更稳定的内边距
- 轻微渐层可保留，但只在低饱和范围内

这里允许有一定色彩差异，因为命中 / 未命中本身就是语义层强信息。

## 实现边界

仍然收敛在以下文件内：

- `frontend/src/components/config-panel/InspectorCards.tsx`
- `frontend/src/components/config-panel/index.tsx`

必要时可新增少量局部样式常量，但不引入新的全局主题层。

不应改动：

- `presentation.ts` 的数据语义
- `displayValues.ts` 的行为
- 表单字段渲染逻辑
- 节点配置保存逻辑

## 验证重点

本轮主要验证视觉细化，而不是功能变化：

1. 头部浅雾蓝方案是否比当前版本更鲜明
2. 去掉 `T` 后头部是否更完整
3. 输入框 / 下拉框是否明显脱离默认 Ant Design 质感
4. 桌面端窄栏下是否仍然稳定可读
5. `TAGGER` 之外的节点头部是否不会过度设计

## 结论

本轮 refinement 明确采用：

- `B` 方向浅雾蓝头部
- 去掉右侧 `T` 标识
- 保留强识别结构
- 将输入和下拉整体拉向更重的 iOS 风格

同时继续坚持最高约束：

- 只做 UI 细化
- 现有功能不能改坏
