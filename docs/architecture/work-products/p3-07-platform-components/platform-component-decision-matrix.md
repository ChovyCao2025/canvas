# Platform Component Decision Matrix

Date: 2026-06-05

Status: P3-07 planning artifact. One component may be accepted for proof; this does not approve production rollout.

## Decision Rules

- Adopt no component without a validated P0, P1, or P2 capability gap.
- Prefer a local interface before business code depends on a vendor API.
- Prefer one proof-of-value at a time.
- Treat component choices as platform decisions, not domain or service boundaries.

## Selected Proof

Redisson is the only component accepted for proof in this iteration. The proof is limited to a local distributed-coordination interface and one low-risk lease use case. It must not replace all Redis usage, rate limiting, scheduler ownership, or execution hot-path coordination without a follow-up ADR.
