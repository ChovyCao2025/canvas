# Architecture Work Products

This folder contains supporting artifacts produced by completed architecture packages. These files are not standalone backlog items; they provide inventories, decision matrices, operating models, test plans, and promotion gates referenced by the archived specs and plans.

## Folders

- `audits/`: audit notes used to validate plan handoff quality.
- `p3-01-platform-evolution/`: platform-evolution promotion gate and governance checklist.
- `p3-02-service-boundaries/`: domain map and contract inventory for service decomposition decisions.
- `p3-03-data-platform/`: source inventory, POC plan, and contract governance for the data-platform slice.
- `p3-04-multi-datasource/`: datasource ownership, transaction boundary, and migration planning artifacts.
- `p3-05-runtime-model/`: WebFlux/MVC migration inventory and first-slice plan.
- `p3-06-k8s-platform/`: K8s operating model.
- `p3-07-platform-components/`: production component decision matrix and abstraction plan.
- `p3-08-wecom-scrm/`: WeCom SCRM implementation slice, integration boundary, and test plan.
- `p3-09-platform-primitives/`: identity, event, tenant, and engine/web boundary platform contracts.

## Status

Completed spec and plan packages are archived under `../archive/completed/`. Reviewed source packages remain under `../reviewed-packages/` for traceability. Historical source reviews remain under `../archive/`.
