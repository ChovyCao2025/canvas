## 1. 问题描述

### 1.1 现状

任何人可直接发布画布，无同行验证。

### 1.2 痛点

- 误操作导致生产环境中断
- 压力测试环境部署到生产环境

---

## 2. 功能需求

### 2.1 核心功能

1. **发布前审核流程**：
   - 创建「发布变更工单」`ticket_type: release_review`
   - 自动邀请新画布的主要开发者参与 Review
   - Review 通过后（`status: APPROVED`）允许发布

2. **错误分类门控**：
   - CRITICAL 错误（如执行超时、网络故障）锁死画布，禁止发布

3. **管理员强制通过**：`ADMIN` 角色 `override_release` 标志

### 2.2 工作量

| 任务 | 工作量（人天） |
|------|--------------|
| 发布工单模型 | 2 |
| 代码流程拦截 | 2 |
| Review UI 开发 | 3 |
| 测试 + 文档 | 2 |
| **总计** | **9** |
