## 1. 问题描述

### 1.1 现状

排障时 trace 日志与画布节点分离，需要手动在多个工具间切换。

### 1.2 痛点

- 问题排查效率低（找节点 URL → 打开工具体验）

---

## 2. 功能需求

### 2.1 核心功能

1. **Trace 点击跳转链接**：
   - 执行流 trace 中点击失败节点 → 自动打开工具体并选中对应节点
   - 格式：`?node_id=123&execution_id=abc123`

2. **执行日志 Scala 跳转**：
   - 点击日志中的 `node_id: 123` → 自动跳转到节点
