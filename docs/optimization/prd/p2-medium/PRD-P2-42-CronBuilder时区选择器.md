## 1. 问题描述

### 1.1 现状

后端默认 Asia/Shanghai，前端无法指定。

### 1.2 痛点

- 北美/亚太/欧洲团队使用时会出现「时差时间错误」，错过执行窗口

---

## 2. 功能需求

### 2.1 核心功能

1. **CronBuilder 时间区选择器**：
   - UI：选择器 `时区[+]`
   - 支持常用时区：UTC, Asia/Shanghai, America/New_York, Europe/London, Asia/Tokyo
   - 支持完整 IANA 时区列表

2. **Cron 表达式标准化**：
   - 存储：`0 */6 * * *`（无时间区标识）
   - 显示：`时区: Asia/Shanghai, Cron: 0 */6 * * *`（前端展示）
