## 1. 问题描述

### 1.1 现状

人群数据仅通过 Cron 任务每 10 分钟刷新一次。

### 1.2 痛点

- 延迟高：事件发生到人群更新最多 10 分钟
- 糟糕的用户体验：用户刚触发行为，但新人群未更新

### 1.3 竞品对标

| 竞品 | 策略 |
|------|------|
| Braze | 实时人群索引（通过 PropSync 推送） |
| Iterable | Real-time Audience Builder |

---

## 2. 功能需求

### 2.1 核心功能

1. **事件驱动刷新**：监听用户行为事件，推送到 Redis（`user_event:{userId}`）
2. **Redis 订阅**：人群服务订阅 Redis channel `user_events:update`
3. **人群异步计算**：消费到事件后异步更新人群（少于 10s）

### 2.2 工作量

| 任务 | 工作量（人天） |
|------|--------------|
| 事件推送逻辑 | 2 |
| Redis 订阅逻辑 | 2 |
| 异步更新队列 | 2 |
| 测试 + 文档 | 2 |
| **总计** | **8** |
