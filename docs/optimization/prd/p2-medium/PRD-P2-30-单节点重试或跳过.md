## 1. 问题描述

### 1.1 现状

执行失败后只能重试整个画布或查看日志。

### 1.2 痛点

- 单个节点超时（如 API 调用超时）导致整个画布失败，无细粒度控制

---

## 2. 功能需求

### 2.1 核心功能

1. **节点级重试策略**：
   - 配置项：`node.retry.policy: { retries: 3, backoff: constant }`
   - 重试失败后，可以选择「跳过节点」或「标记错误并继续」

2. **节点级日志跳转**：
   - 执行失败时，UI 点击「详情」跳转到对应节点日志页面
