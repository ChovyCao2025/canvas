## 1. 问题描述

### 1.1 现状
所有列表页均无行选择功能。用户需要逐个删除、导出或编辑数据，重复劳动严重。

### 1.2 痛点
操作效率低下：逐个点击操作耗时且易出错；批量操作缺失：无法一次性选择多个条目进行操作；错误操作率高：误点击导致部分条目误删除。

### 1.3 竞品对标
| 竞品 | 能力 |
|------|------|
| Excel | 支持多选、批量删除/复制/移动 |

---

## 2. 目标与价值

### 2.1 用户故事
> 作为 系统管理员，我希望 选择多个列表项并执行批量操作，以便 大幅提升操作效率，减少人工错误，实现规模化数据管理

### 2.2 成功指标
- 至少 5 个列表页支持行选择 + 批量操作
- 支持 Ctrl/Cmd + 点击多选行，Shift + 点击范围选择

---

## 3. 功能需求
1. Table 组件行选择列 + 选中行高亮
2. 批量操作按钮（删除、导出、启停）
3. 支持右键菜单批量操作 + 键盘快捷键

---

## 4. 非功能需求
- **性能要求**：批量删除 100 条数据响应时间 < 2s
- **安全要求**：批量操作需二次确认，后端需权限验证
- **可用性要求**：支持 Tab 键导航到行选择框，支持 Delete/Backspace 删除

---

## 5. 验收标准
- [ ] 至少 3 个列表页支持行选择
- [ ] 选中多行后显示批量操作按钮
- [ ] 支持 Ctrl+A 全选，选中数量实时显示
