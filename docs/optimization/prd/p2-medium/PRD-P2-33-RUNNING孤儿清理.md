## 1. 问题描述

### 1.1 现状

重启后 RUNNING 执行记录永远保持 RUNNING。

### 1.2 痛点

- 数据库表堆积过量 RUNNING 记录，影响查询性能
- 无法区分「正常 RUNNING」和「孤儿 RUNNING」

---

## 2. 功能需求

### 2.1 核心功能

1. **孤儿清理策略**：
   - Cron 定时清理最近 24 小时无活动的 RUNNING 记录
   - 写入清理日志（`status: ORPHAN_CLEANED`）

2. **运行心跳**：
   - 节点执行每隔 5 分钟触发心跳（`last_heartbeat` 更新）
   - 心跳超时判定为孤儿
