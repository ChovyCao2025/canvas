## 1. 问题描述

### 1.1 现状

紧急回滚需要分两步操作：先 kill，再 rollback。

### 1.2 痛点

- 故障时操作超时累积，响应时间窗口极短

---

## 2. 功能需求

### 2.1 核心功能

1. **回滚快捷方式**：
   - 单个按钮 `一键回滚` → 自动 kill current executions → 回滚到上一个版本 → 重新发布
   - 权限：`ADMIN` 角色强制通过

2. **回滚确认对话框**：
   - 确认回滚后 10 秒内执行
   - 提供「回滚原因」填写（用于审计）

| 测试 + 文档 | 1 |
| **总计** | **6** |
