## 1. 问题描述

### 1.1 现状

画像中途被修改，导致执行期间数据不一致。

### 1.2 痛点

- 营销节点需要访问「用户当前标签」，但中途标签变动可能导致营销策略错误

---

## 2. 功能需求

### 2.1 核心功能

1. **快照机制**：
   - 执行开始时快照保管：`execution_snapshot: { userId, tagName: 'VIP', timestamp: 123456789 }`
   - 中途标签修改不影响执行节点读取方向

2. **快照恢复**：
   - 节点抛出 `SnapshotExpiredException` → 回滚到快照状态
