## 1. 问题描述

### 1.1 现状

仅记录原始错误字符串，无 `errorCategory` 字段。

### 1.2 痛点

- 无法按错误类型聚合（如「网络错误 50/0」「验证失败 30/0」）
- 无法自动化告警（如仅告警「网络错误」不告警「验证失败」）

---

## 2. 功能需求

### 2.1 核心功能

1. **错误分类枚举**：
   - `NETWORK_ERROR`: 超时、DNS 解析失败
   - `VALIDATION_ERROR`: 参数校验失败
   - `BUSINESS_ERROR`: 业务逻辑失败（如余额不足）
   - `SYSTEM_ERROR`: JVM 崩溃

2. **错误聚合分析**：
   - API 返回 `error:{ category, code, message }`
   - UI 「执行失败分析」按 category 筛选
