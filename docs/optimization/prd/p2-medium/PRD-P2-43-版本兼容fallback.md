## 1. 问题描述

### 1.1 现状

删除节点类型后执行直接报错，无降级方案。

### 1.2 痛点

- 灰度发布中遇到版本不兼容执行，系统崩溃

---

## 2. 功能需求

### 2.1 核心功能

1. **Dependency Graph**：
   - 记录画布依赖：`node_type: "api_call"` → `depends_on: ["openai_api_key"]`
   - 当节点类型被删除时，标记为 `DEPRECATED` 并生成 fallback 节点

2. **Deprecation 通告**：
   - UI 右上角弹出 `v2.2.0` 依赖废弃提示
   - 提供「自动迁移」按钮（可选迁移到新节点类型）
