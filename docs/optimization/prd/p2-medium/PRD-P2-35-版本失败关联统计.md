## 1. 问题描述

### 1.1 现状

无法快速判断「新版本是否导致更多失败」。

### 1.2 痛点

- 版本变更后无法快速回滚，必须人工分析所有执行记录

---

## 2. 功能需求

### 2.1 核心功能

1. **版本关联统计**：
   - 记录每一次画布发布的 `published_version_id`
   - 按版本统计失败率（如 v2.1: 10%失败, v2.2: 25%失败）

2. **失败原因对比**：
   - 对比两次发布的失败原因（v2.1 网络错误为主 → v2.2 验证错误为主）

| 统计分析 UI | 3 |

| 测试 + 文档 | 1 |
| **总计** | **6** |
