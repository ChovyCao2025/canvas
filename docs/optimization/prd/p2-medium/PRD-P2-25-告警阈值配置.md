## 1. 问题描述

### 1.1 现状

仅有事后通知，无主动监控阈值。

### 1.2 痛点

- CPU 占用 95% 时未告警，导致内存溢出
- 日志文件增长超过 1GB 未触发告警

---

## 2. 功能需求

### 2.1 核心功能

1. **监控指标**：
   - `cpu_usage`, `memory_usage`, `disk_usage`
   - `active_executions`, `queue_size`
   - `error_rate`

2. **告警策略**：
   - `critical`: 系统 CPU > 90% → 邮件通知 ADMIN
   - `warning`: 日志大小 > 1GB → 邮件通知 DevOps
   - `info`: 活跃执行数 > 3000 → 企业微信通知

3. **告警轮转**：
   - 重复告警：每 5 分钟最多发送 1 次
   - 告警恢复：记录恢复日志
