## 1. 问题描述

### 1.1 现状

只有单用户路径视图，无法查看「10,000 用户平均通过此节点的概率为 80%」。

### 1.2 痛点

- 营销人员无法定位高流失的节点（如「点击发送工单」后 50% 的用户流失）

### 1.3 竞品对标

| 竞品 | 能力 |
|------|------|
| Braze | Journey Builder 提供聚合模式（Funnel, Series） |
| Iterable | Campaign Analytics 支持多用户路径统计 |

---

## 2. 功能需求

### 2.1 核心功能

1. **路径聚合视图**：
   - Funnel 模式：`事件 A → 事件 B → 事件 C`
   - Series 模式：10,000 用户中，80% 一天内收到 3 次推送

2. **按人群筛选**：仅分析「付费客户」的路径

3. **按时间段筛选**：仅分析「本月」

### 2.2 工作量

| 任务 | 工作量（人天） |
|------|--------------|
| 路径聚合逻辑 | 4 |
| 交互式图表开发 | 3 |
| 按人群筛选 | 2 |
| 测试 + 文档 | 2 |
| **总计** | **11** |
