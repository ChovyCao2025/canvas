## 1. 问题描述

### 1.1 现状

全中文硬编码，无 i18n。

### 1.2 痛点

- 国际化本地化扩展成本高（新增语言需要修改所有代码）

---

## 2. 功能需求

### 2.1 核心功能

1. **i18n 配置集成**：
   - 使用 `react-i18next` 集成国际化框架
   - 配置语言支持：中文（zh-CN）、英文（en-US）

2. **前端工程化**：
   - 分离 UI 文本为 `locales/zh-CN.json`, `locales/en-US.json`
   - 动态语言切换（切换后自动刷新 UI）

### 2.2 工作量

| 任务 | 工作量（人天） |
|------|--------------|
| i18next 集成 | 2 |
| UI 文本分离 | 2 |
| 语言切换 | 1 |
| 测试 + 文档 | 1 |
| **总计** | **6** |
