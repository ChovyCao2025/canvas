## 1. 问题描述

### 1.1 现状

仅限制深度（DAG 层级），不限制宽度（节点总数）。

### 1.2 痛点

- 无限制宽度可导致浏览器渲染卡死、Dagre 布局计算超时、数据库内存溢出

---

## 2. 功能需求

### 2.1 核心功能

1. **画布节点数限制**：
   ```yaml
   canvas.node_limit: 50
   canvas.node_limit_warning: 40  # 40 个节点弹出「接近上限」提醒
   ```

2. **节点数统计 UI**：
   - 画布工具栏显示 `节点: 45/50`，形成「节点数倒计时」
