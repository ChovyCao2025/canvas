## 1. 问题描述

### 1.1 现状

当前仅支持 `OFFLINE_BATCH`，无法选择 ONLINE人群策略。

### 1.2 痛点

- ONLINE 策略可实时读取 Redis 中活跃用户，延迟 < 100ms
- OFFLINE 策略延迟高（5-10 秒）

---

## 2. 功能需求

### 2.1 核心功能

1. **人群策略枚举**：`ONLINE`, `OFFLINE_BATCH`
2. **按人群类型过滤**：`filterBy: { type: "ONLINE", limit: 10000 }`
