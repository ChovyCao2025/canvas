## 1. 问题描述

### 1.1 现状

仅有 in-app WebSocket，平台宕机后无法通知。

### 1.2 痛点

- 平台大面积宕机（如服务器机房宕机），运维人员无渠道通信

---

## 2. 功能需求

### 2.1 核心功能

1. **灾难通信渠道**：
   - 电话树：通过云端电话树自动拨号循环拨打（call tree）
   - 钉钉集群告警：钉钉安全告警机器人（电话/短信）
   - 邮件风暴：一次性向所有管理员发送 `@所有人` 邮件

2. **多渠道优先级**：
   ```yaml
   disaster_channels:
     - type: "call_tree"
       recipients: [manager_a, manager_b, manager_c]
       fallback: "slack"  # 电话失败后 fallback 到 Slack
     - type: "slack"
       channel: "#alert"
       recipients: TEAM_ADMIN
   ```
