## 1. 问题描述

### 1.1 现状

- 模板只能创建，不可在线编辑或删除
- 鼓励建立复制 （Copy Template） 微服务但未普及
- 错误创建时无法修正，只能删除重建

### 1.2 痛点

- 命名与配置错误无法热修复，需流程重建
- 历史模板堆积，未轮轠除或归档
- 更换代际重试依赖多个系统

---

## 2. 目标与价值

### 2.1 用户故事

> 作为 画布创建者，我希望 终端编辑模板并按需删除，以便 修正初始错误、清理历史模板降低维护压力。

### 2.2 成功指标

- 模板预览 与编辑保存单次延迟 < 500ms
- 编辑模板后基于该模板创建的画布不自动污染改动
- 历史模板每月归档率 > 80%

### 2.3 不做会怎样

- 无法修正错误配置，只好重建模板
- 历史模板继续占用空间和心智成本
- 无法快速测试而反复重建影响效率

---

## 3. 功能需求

### 3.1 核心功能

1. **模板预览** — 离线查看节点结构与参数（不可直接编辑）
2. **在线编辑** — 基于模板创建画布后可云状态下在线编辑
3. **复制模板** — 将模板 Clone 为新模板（原有访问能力保留）
4. **删除模板** — 按现有 cascade 子画布判断（可选删除）
5. **版本控制** — 模板版本递增，新版不覆盖旧版

### 3.2 详细描述

- UI 布局：画布编辑器底部「模板」标签页显示模板归属
- API 端点：
  - `PUT /api/v3/templates/{id}` — 更新模板（只允许 DRAFT 状态）
  - `DELETE /api/v3/templates/{id}` — 删除模板（基于模板的子画布列表判断）
  - `POST /api/v3/templates/{id}/copy` — 复制模板
- 编辑保护：
  - 只允许编辑「基于模板创建的画布」（已发布画布只读）
  - 若现有子画布中存在未完成 Execute，编辑应提示选中影响范围

### 3.3 交互流程

1. 在画布列表中选择「基于模板创建的画布」
2. 点击「编辑模板」→ 前端切换为「模板编辑模式」
3. 后端只查 template_id 到 canvas 的映射，返回对应配置
4. 编辑后必须保存才能生效

---

## 4. 非功能需求

- **性能要求**: 模板读取需使用缓存，编辑变更后清除对应缓存
- **安全要求**: 删除前必须校验无进行中的执行
- **可用性要求**: 编辑界面提供「撤销次次」能力

---

## 5. 验收标准

- [ ] 模板状态只有在 DRAFT 时可编辑
- [ ] 编辑模板后 Snapshot 正确
- [ ] 删除模板前存在执行的任务不做删除
- [ ] 复制后模板 ID 变更，内容完全一致

---

## 6. 技术建议

### 6.1 涉及模块

- 后端 TemplateController（增加 PUT / DELETE / COPY）
- 缓存层（TemplateEntity 缓存）

### 6.2 技术要点

- 使用模板 ID 作为 key，模板内容存为 JSON
- 删除时需扫描子画布表 `canvas` 中 `template_id = {id}` 的情况
- 基于模板创建的画布可在 `canvas.template_id` 字段关联

### 6.3 预估工作量

- API 开发：3 人天
- 前端 UI：2 人天
- 缓存/审计集成：1 人天

---

## 7. 依赖与风险

### 7.1 前置依赖

- 当前 Template 模型缺失，创建 V2

### 7.2 风险

- 删除模板可能影响未完成执行链路
- 编辑未保存的改动导致已发布的画布不一致
