# PRD-P1-17-实时协作在线状态

> 本文档为营销画布平台产品需求文档标准模板，每项缺项对应一份独立 PRD。

---

## 基本信息

| 字段 | 内容 |
|------|------|
| **PRD编号** | PRD-P1-17 |
| **需求名称** | 实时协作在线状态 |
| **优先级** | P1 |
| **所属类别** | 运营工作流 |
| **提出日期** | 2026-05-31 |
| **来源** | BMAD 产品设计审查报告 |

---

## 1. 问题描述

- 仅基于乐观锁冲突检测，缺乏实时协作模式的 lead 真正在线冷+移交保护机制

---

## 2. 目标与价值

### 2.1 用户故事

> 作为 编辑者，我希望 实时看到其他协作者是否正在编辑画布，以便 决定是否接力或等待，避免内容覆盖。

---

## 3. 功能需求

### 3.1 核心功能

1. **用户在线检测** - 踦引用 ping/pong 机制检测空闲时间（如 15s 无操作判定为离线）
2. **编辑锁管理** - curOwner 锁防恢复；操作链防正在变更泄漏；交接检测防变频冲突
3. **UI 展示** - 在线状态可视化（头像旁显示在线图标）
4. **交接确认** - 用户 A 点击「切换负责人」时通知用户 B 是否接受

---

## 4. 非功能需求

性能要求使用多配置项外 + 实时 ping/pong + UI 轻量依赖；使用线程安全 ConcurrencyMap 保持状态不抖动

---

## 5. 参考资料

- BMAD 产品设计审查报告 (2026-05-31)
