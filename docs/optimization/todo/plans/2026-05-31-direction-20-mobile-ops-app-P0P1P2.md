# 方向⑳：移动端运营App — 功能清单

> 定位：从"仅PC端"升级为"移动端运营App"——运营人员随时查看效果+审批+紧急操作
> 策略评估：无移动端，运营人员离线时无法操作；10-15人月可完成核心
> 竞品对标：Braze Mobile Dashboard、HubSpot Mobile、Marketo Mobile
> 建议：**P3远期**，PC端体验优先，移动端是锦上添花

---

## 现状盘点

| 功能 | 实现程度 | 核心现有代码 | 差距 |
|------|----------|-------------|------|
| PC端管理后台 | **完整** | React前端+后端API | 管理后台完整 |
| 响应式布局 | **部分** | 仅首页有响应式 | 其他页面无移动端适配 |
| 移动端 | **不存在** | — | 完全缺失 |
| Push通知 | **部分** | 可发Push，但无App接收 | 发送能力有，接收App无 |

---

## 功能清单

### P0 — 核心查看与操作

---

#### 1. 移动端效果看板 [中复杂度 | 2.0人月]

**需补齐**：

| 页面 | 描述 | 核心组件 |
|------|------|---------|
| 首页 | 今日核心指标+趋势 | 指标卡片+迷你折线图 |
| 画布列表 | 我的画布列表+状态 | 列表+状态标签+筛选 |
| 画布详情 | 画布效果指标+节点漏斗 | 漏斗图+指标卡片 |
| 效果趋势 | 7天/30天效果趋势 | 折线图 |

**技术方案选择**：

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| 原生App | 体验最佳 | 开发成本高(iOS+Android) | 远期 |
| React Native | 跨平台+复用前端技能 | 性能略差 | **近期** |
| PWA | 无需安装+开发成本低 | 能力受限(无Push) | 备选 |
| 小程序 | 中国市场友好+无需安装 | 仅微信生态 | **中国市场优先** |

---

#### 2. 移动端审批 [低复杂度 | 1.0人月]

**需补齐**：

| 子功能 | 描述 |
|--------|------|

| 审批详情 | 画布概览+人群预估+触达渠道 |

#### 3. 移动端紧急操作 [低复杂度 | 1.0人月]

**需补齐**：

| 操作 | 描述 |
|------|------|

### P1 — 通知与协作

---

#### 4. Push通知 [中复杂度 | 1.5人月]

**需补齐**：

| 通知类型 | 描述 | 优先级 |
|---------|------|--------|
| 审批通知 | 有新审批项 | 高 |
| 执行失败 | 画布执行失败 | 高 |

| 审批驳回 | 我的画布被驳回 | 中 |
| 日报 | 每日效果摘要 | 低 |

## 执行顺序

```
Sprint 1 (P0-看板): 移动端效果看板 — 2.3人月
  → 产出：核心指标+画布列表+效果趋势

Sprint 2 (P0-操作): 审批+紧急操作 — 2.4人月
  → 产出：审批通过/驳回+暂停/恢复画布

Sprint 3 (P1-通知): Push通知+评论 — 2.3人月
  → 产出：审批/告警通知+画布评论

Sprint 4 (P2-高级): 画布预览+语音 — 2.3人月
  → 产出：流程图预览+语音指令
```

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 小程序审核 | 微信小程序审核周期 | 预留审核时间+Web版兜底 |
| 移动端体验 | 小屏幕操作不便 | 核心操作简化+大按钮 |
| Push权限 | 用户未开启Push通知 | 首次使用引导+降级为站内通知 |
| 离线场景 | 网络不稳定时操作失败 | 离线缓存+自动重试 |
