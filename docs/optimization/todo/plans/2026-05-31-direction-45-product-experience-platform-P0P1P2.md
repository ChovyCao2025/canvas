# 方向㊺：产品体验与分析平台(PXP) — 功能清单

> 定位：从"营销触达"延伸到"产品内体验"——应用内引导(Product Tours)+功能采用分析+应用内消息+产品分析+用户反馈
> 策略评估：Pendo/Appcues/Userpilot验证了"产品体验层"的独立价值。营销的边界正延伸到产品内——用户在产品内的行为是最强的意图信号
> 竞品对标：Pendo(产品分析+引导+反馈)、Appcues(无代码Product Tours)、Userpilot(产品体验+用户引导)、Intercom(客服+产品导览)
> 建议：**P2建议做**，依赖㉜事件SDK+⑬画像+㉞个性化引擎成熟后启动，产品体验是营销的下一个触达阵地

## 功能清单

### P0 — 产品数据采集与分析 [2.0人月]

| 子功能 | 描述 |
|--------|------|
| 功能采用分析 | 每个功能的DAU/MAU/采用率/留存率/使用时长/使用频率 |
| 用户路径分析 | 用户在产品内的行为路径(Sankey图)+关键转化漏斗 |
| 产品留存 | 新用户Day1/Day7/Day30留存+功能留存+分群留存对比 |
| 用户分群(产品维度) | 按产品行为分群(活跃用户/即将流失/高级用户) |

### P1 — 应用内引导与消息 [1.5人月]

| 子功能 | 描述 |
|--------|------|
| 产品导览(Tours) | 无代码创建多步骤产品引导(Tooltips/Modals/Slideouts) |
| 新用户Onboarding | 新用户自动触发Onboarding流程(欢迎→核心功能引导→首次Aha Moment) |
| 功能介绍 | 新功能上线→自动弹窗介绍+引导试用 |
| 应用内消息 | 产品内嵌入消息(公告/提示/升级提醒)+用户行为触发 |

### P2 — PXP+营销联动 [1.0人月]

| 子功能 | 描述 |
|--------|------|
| 产品行为→营销触发 | 用户尝试了高级功能但未升级→触发营销画布(Sales+优惠) |
| 产品留存→客户成功 | 使用频率下降→自动触发CSM跟进/使用技巧推送 |
| 产品反馈→产品改进 | 功能评分+反馈收集→反馈→产品工单(对接㊷) |
