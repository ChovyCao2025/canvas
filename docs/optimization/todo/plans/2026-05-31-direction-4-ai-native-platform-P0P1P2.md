> 定位：不是在画布里加AI节点，而是让AI成为画布的操盘手

## 功能清单

### Phase 1 (0-6月) AI辅助 — 8.0人月

---

#### 1. 流失预测模型 [高复杂度 | 3.0人月]

**目标**：预测用户在未来N天内流失的概率

**模型选型**：

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| 规则+统计 | 无需训练数据，快速上线 | 准确率低（~60%） | MVP阶段 |
| XGBoost/LightGBM | 准确率高（~80%），可解释 | 需特征工程+训练数据 | 主力方案 |
| 深度学习(LSTM) | 时序建模强 | 训练复杂，推理慢 | 远期 |

**特征工程**：
| 特征组 | 特征 | 来源 |
|--------|------|------|
| 活跃度 | 最近7/14/30天登录次数、页面浏览数 | 事件日志 |
| 交易 | 最近7/14/30天购买次数/金额、客单价趋势 | 订单数据 |
| 互动 | 邮件打开率/点击率、Push响应率 | 渠道回执 |
| 画像 | 会员等级、注册天数、生命周期阶段 | 用户画像 |
| 行为 | 最近搜索/收藏/加购次数 | 行为事件 |

**与画布联动**：
```
1. 定时任务每日计算流失概率
2. 流失概率>0.7的用户打上 churn_risk=HIGH 标签
3. AUDIENCE_TRIGGER 圈选 churn_risk=HIGH 人群
4. 启动"流失挽回"画布
```

#### 3. 内容个性化推荐 [中复杂度 | 2.0人月]

**目标**：根据用户画像和行为，推荐最可能转化内容

**方案**：
| 阶段 | 方案 | 效果 |
|------|------|------|
| MVP | 基于规则（用户标签→内容分类映射） | 召回率低 |
| V2 | 协同过滤（相似用户喜欢的内容） | 中等 |
| V3 | 深度学习推荐模型 | 高 |

**NodeHandler**：
- `AI_RECOMMEND` — AI推荐节点（替代现有RecommendationHandler stub）
- 输出：推荐内容列表+推荐理由+置信度

---

#### 4. AI_LLM节点实化 [中复杂度 | 1.0人月]

**描述**：将AiNextBestActionHandler从stub升级为真实LLM调用

**技术方案**：
```java
public class AiNextBestActionHandler extends AbstractNodeHandler {
    private final LlmClient llmClient;  // OpenAI/Claude/本地模型

    public Mono<NodeResult> executeAsync(NodeConfig config, ExecutionContext ctx) {
        String prompt = buildPrompt(config, ctx);  // 从上下文构建prompt
        return llmClient.complete(prompt)
            .map(response -> parseAction(response))
            .onErrorResume(e -> Mono.just(NodeResult.ok(
                fallbackNodeId,
                Map.of("action", config.getString("fallbackAction"),
                       "aiFallbackUsed", true))));
    }
}
```

**LLM接入方式**：
| 方案 | 优点 | 缺点 |
|------|------|------|
| OpenAI API | 能力强 | 数据出境合规风险 |
| Claude API | 安全性好 | 同上 |
| 本地模型(Ollama) | 数据不出境 | 能力弱，需GPU |

### Phase 2 (6-18月) AI自动化 — 11.5人月

---

#### 5. CLV预测 [高复杂度 | 3.0人月]

**目标**：预测用户未来12个月的生命周期价值

**模型**：BG/NBD + Gamma-Gamma（经典CLV模型）或ML回归

**应用**：
- 高CLV用户优先服务
- 低CLV用户降低触达成本
- CLV作为RFM分层中的M指标

---

#### 6. 渠道偏好预测 [中复杂度 | 2.0人月]

**目标**：预测用户最可能响应的渠道

**方案**：多分类模型（EMAIL/SMS/PUSH/WECHAT/IN_APP），基于历史回执数据

---

#### 7. 发送优化 [中复杂度 | 2.5人月]

**目标**：自动优化发送策略（渠道组合+频次+内容）

**技术方案**：多臂老虎机(Multi-Armed Bandit) + Thompson Sampling

---

#### 8. 异常检测 [中复杂度 | 2.0人月]

**目标**：自动检测营销指标异常（打开率骤降/退订率飙升/渠道失败率上升）

**方案**：统计过程控制(SPC) + 3-sigma规则

---

#### 9. 自动A/B测试 [中复杂度 | 2.0人月]

**目标**：自动创建A/B实验、自动判断胜者、自动切换

---

### Phase 3 (18-36月) AI Agent — 18.5人月

---

#### 10. NL画布生成 [极高复杂度 | 6.0人月]

**目标**：用自然语言描述营销需求，自动生成画布

**示例**：
```
用户输入："给30天没购买的金卡会员发一张50元优惠券，如果3天内没使用就发短信提醒"

AI生成：
  AUDIENCE_TRIGGER → [30天未购买+金卡会员]
  → COUPON(50元代金券)
  → WAIT(3天)
  → IF_CONDITION(券是否使用?)
     → YES: 标记转化
     → NO: SEND_SMS(提醒)
```

**技术方案**：
- LLM fine-tune：用画布JSON结构训练模型
- 生成后人工确认+编辑

---

#### 11. 目标驱动编排 [极高复杂度 | 5.0人月]

**目标**：运营只输入业务目标（如"提升复购率5%"），AI自动设计+执行+优化画布

---

#### 12. 跨画布优化Agent [高复杂度 | 4.5人月]

**目标**：Agent监控所有画布效果，自动调整频次/人群/内容/时机

---

#### 13. 自改进模型 [高复杂度 | 3.0人月]

**目标**：模型基于执行效果自动retrain，持续提升预测准确率

## 不推荐独立启动的原因

| 风险 | 说明 |
|------|------|
| 数据依赖 | AI效果依赖历史数据量，早期客户数据不够 |
| 人才需求 | 需ML工程师+数据科学家，当前团队无此能力 |
| 周期长 | 37人月+3年，融资不支持 |
| Customer.io领先 | Cowork AI Agent(MCP协议)已领先6-12个月 |

**建议**：作为方向②私域中台的Phase 3(18-36月)，在数据积累到一定程度后启动。
