# 方向⑰：运营工作台 — 功能清单

> 定位：从"散落的功能页面"升级为"统一运营工作台"——运营日历+任务看板+效果看板+快捷操作
> 策略评估：当前运营人员需在多个页面间跳转，无统一入口；4-6人月可完成核心
> 竞品对标：HubSpot Marketing Calendar、Braze Dashboard、Iterable Campaign Overview
> 建议：**P2建议做**，运营体验提升，团队>3人时刚需

---

## 现状盘点

| 功能 | 实现程度 | 核心现有代码 | 差距 |
|------|----------|-------------|------|
| 画布列表 | **完整** | CanvasController.list() | 有列表，无运营视角 |
| 画布统计 | **部分** | CanvasStatsController(stats/funnel/trend) | 有统计，缺运营聚合 |
| 首页看板 | **部分** | 前端HomePage | 仅展示画布数量，缺运营指标 |
| 运营日历 | **不存在** | — | 完全缺失 |
| 任务看板 | **不存在** | — | 完全缺失 |
| 快捷操作 | **不存在** | — | 无一键暂停/发布/复制 |
| 通知中心 | **不存在** | — | 无运营通知聚合 |

### 关键洞察

运营人员的日常工作流：
1. 早9点：看昨日效果数据 → 当前需逐个画布查看
2. 规划：本周要上哪些活动 → 无日历视图
3. 执行：创建/修改画布 → 需在列表中翻找
4. 监控：活动运行是否正常 → 无异常告警
5. 收尾：活动结束暂停画布 → 需逐个操作

缺的是一个**统一入口**，把散落的功能聚合到运营工作台。

## 功能清单

### P0 — 运营工作台核心

---

#### 1. 运营仪表盘 [中复杂度 | 1.5人月]

**现状**：首页仅展示画布数量

**需补齐**：

| 卡片 | 描述 | 数据来源 |
|------|------|---------|
| 今日触达 | 今日消息发送数 | Redis实时计数 |
| 活跃画布 | 当前运行中的画布数 | CanvasDO.status=RUNNING |
| 待审批 | 等待我审批的画布数 | canvas_approval(PENDING) |
| 本周执行 | 本周画布执行总次数 | CanvasExecutionStatsDO |
| 触达趋势 | 最近7天触达量趋势 | canvas_metrics_daily |
| 画布状态分布 | 各状态画布数饼图 | CanvasDO按status分组 |
| 异常告警 | 最近24小时失败/超时画布 | CanvasExecutionDO(FAILED/TIMEOUT) |

**仪表盘布局**：

```
┌──────────┬──────────┬──────────┬──────────┐
│ 今日触达  │ 活跃画布  │ 待审批    │ 本周执行  │
│  12,350  │    8     │    3     │  1,256  │
├──────────┴──────────┴──────────┴──────────┤
│              触达趋势(7天)                   │
│        📈 折线图                           │
├──────────────────┬────────────────────────┤
│   画布状态分布     │     异常告警            │
│   🥧 饼图        │  ⚠ 画布#5 超时率30%     │
│                  │  ⚠ 画布#8 退订率5%      │
└──────────────────┴────────────────────────┘
```

---

#### 2. 运营日历 [中复杂度 | 1.5人月]

**现状**：无日历视图

**需补齐**：

| 子功能 | 描述 |
|--------|------|
| 月视图 | 画布执行时间在日历上展示 |
| 周视图 | 按周展示画布执行计划 |
| 日视图 | 按天展示详细执行计划 |
| 画布标记 | 日历上标记画布执行(颜色=渠道/状态) |
| 拖拽调整 | 拖拽修改定时画布执行时间 |
| 冲突检测 | 同一时段画布执行冲突告警 |
| 筛选 | 按渠道/状态/创建人筛选 |

**日历数据来源**：

| 日历事件 | 来源 | 时间字段 |
|---------|------|---------|
| 定时触发画布 | CronSchedule+canvas_schedule | nextFireTime |
| 画布开始执行 | CanvasExecutionDO | createdAt |
| 画布执行完成 | CanvasExecutionDO | updatedAt(STATUS=SUCCESS) |
| 审批到期 | canvas_approval | timeout_at |

### P1 — 任务与通知

---

#### 3. 任务看板 [中复杂度 | 1.0人月]

**现状**：无任务看板

**需补齐**：

| 列 | 描述 |
|-----|------|
| 待处理 | 我需要操作的事项(待审批/失败重试/配额告警) |
| 进行中 | 我正在进行的操作(编辑中的画布/审批中) |
| 已完成 | 最近完成的操作 |

**任务类型**：

| 任务 | 描述 | 操作 |
|------|------|------|
| 待审批 | 画布等待我审批 | 通过/驳回 |
| 审批驳回 | 我的画布被驳回 | 查看意见→编辑 |
| 执行失败 | 画布执行失败 | 查看原因→重试 |
| 配额告警 | 租户配额接近上限 | 查看用量→升级 |
| 定时到期 | 定时画布即将执行 | 查看详情→暂停 |
| 评论@我 | 有人@我评论 | 查看评论→回复 |

---

#### 4. 通知中心 [中复杂度 | 1.0人月]

**现状**：NotificationRecipientService存在但功能有限

**需补齐**：

| 子功能 | 描述 |
|--------|------|
| 站内通知 | 应用内通知列表(未读/已读) |
| 通知类型 | 审批/执行/告警/评论/系统 |
| 通知偏好 | 用户可配置接收哪些通知 |
| 飞书/邮件推送 | 重要通知推送到飞书/邮件 |
| 批量已读 | 一键标记全部已读 |

### P2 — 快捷操作与效率

---

#### 5. 快捷操作 [低复杂度 | 0.5人月]

**描述**：常用操作的一键入口

| 操作 | 描述 |
|------|------|
| 批量暂停 | 一键暂停多个画布 |
| 批量恢复 | 一键恢复多个暂停画布 |

---

#### 6. 运营模板 [低复杂度 | 0.5人月]

**描述**：预置运营场景模板

| 模板 | 描述 | 包含节点 |
|------|------|---------|
| 日常促活 | 每日促活消息 | 定时触发→人群筛选→消息发送→效果统计 |
| 新人引导 | 新用户7天引导 | 事件触发→条件分流→邮件/Push→延迟→再次触达 |
| 流失挽回 | 30天未活跃挽回 | 定时扫描→人群筛选→邮件→优惠券→短信 |
| 节日营销 | 节日大促 | 定时触发→人群→多渠道→A/B测试→效果统计 |
| 生日关怀 | 生日月关怀 | 定时触发→人群→祝福邮件→优惠券 |

## 执行顺序

```
Sprint 1 (P0-仪表盘): 运营仪表盘 — 1.7人月
  → 产出：核心指标卡片+趋势图+异常告警

Sprint 2 (P0-日历): 运营日历 — 1.7人月
  → 产出：月/周/日视图+拖拽调整

Sprint 3 (P1-任务): 任务看板+通知 — 2.2人月
  → 产出：任务列表+站内通知

Sprint 4 (P2-效率): 快捷操作+模板 — 1.2人月
  → 产出：批量操作+运营模板
```

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 仪表盘性能 | 实时指标聚合慢 | Redis缓存+定时刷新 |
| 日历数据量 | 大量画布执行记录 | 按时间范围查询+懒加载 |
| 通知轰炸 | 过多通知降低关注度 | 通知偏好设置+聚合通知 |
