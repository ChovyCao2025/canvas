# 项目演进方向分析（2026-05-31）

## 起点：项目现状

| 维度 | 现状 | 强度 |
|------|------|------|
| DAG编排引擎 | 57个NodeHandler、8种触发器、Disruptor分发、4级Lane、熔断、DLQ、重入 | 很强 |
| CDP | 用户画像+标签+人群+双规则引擎(Aviator/QLExpress)+Bitmap存储 | 中等 |
| 渠道 | 5个Send Handler，但InApp是mock、Coupon/Reach/Recommendation是WireMock | 弱 |
| 分析 | 仅执行KPI和节点漏斗，零行为分析 | 极弱 |
| AI | 1个stub(AI_NEXT_BEST_ACTION) | 零 |
| 合规 | MarketingPolicyService存在但未接入发送链路 | 零 |
| 生产就绪 | 无K8s、无CI/CD、无Redis HA、无审计日志 | 零 |

核心判断：引擎很强但缺少"轮子和方向盘"——DAG引擎是真正的差异化资产，其他全是短板。

---

## 七大演进方向

### 方向①：营销自动化深度（当前定位，继续向右走）

把现在的营销画布做到Braze级别，补齐运营管控层。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| 全局疲劳度策略 | P0 | 中 |
| 归因引擎（末次/首次/线性/衰减） | P1 | 高 |
| 合规门（发布前审核+发送链路拦截） | P0 | 中 |
| 模板市场UI | P1 | 低 |
| 画布版本管理+回滚 | P1 | 中 |
| 运营日历+冲突检测 | P1 | 中 |
| 渠道回执追踪（送达/打开/点击/退订） | P1 | 中 |
| Webhook/事件订阅 | P1 | 中 |
| 批量操作+审批流 | P1 | 低 |
| 智能触达时机 | P2 | 高 |
| 内容管理+模板渲染引擎 | P1 | 中 |

- 优势：当前定位最熟悉，每项能力都有清晰对标
- 风险：画布编排已L2，继续投入边际收益递减；补齐后仍是"中国版Braze"，无差异化护城河
- 适合：有明确客户在等这些功能

### 方向②：私域运营中台（向上走，中国市场特色）

从"触达工具"升级为"用户价值管理平台"，主打企微+小程序+私域。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| 企微SCRM模块（客户管理+群聊运营+朋友圈） | P0 | 高(4-6月) |
| 小程序触达+事件追踪 | P0 | 中(2月) |
| 裂变营销引擎（邀请有礼+拼团+分享解锁） | P1 | 中(2-3月) |
| 生命周期阶段识别+自动化编排 | P1 | 中 |
| 优惠券全生命周期管理 | P1 | 中 |
| 积分/会员等级体系 | P1 | 中 |
| 用户价值分层(RFM/RFE) | P2 | 低 |
| 运营SOP模板 | P2 | 低 |

- 优势：海外竞品结构性无法进入（WeCom API需中国营业执照）；国内竞品画布远弱于我们（20-30节点 vs 57节点）；"DAG级画布+企微原生SCRM"目前无人提供；私域是2020后中国营销最大趋势
- 风险：企微API更新频繁维护成本高；纯私域方向海外不可用
- 适合：目标客户是中国消费品牌/零售/连锁

### 方向③：实时决策引擎（向下走，技术驱动）

用DAG引擎做毫秒级业务决策（风控/推荐/路由/定价），独立于营销场景。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| 轻量决策API（绕过Disruptor+Lane，<50ms SLA） | P0 | 高 |
| 规则引擎强化（决策表+规则集+优先级冲突解决） | P0 | 中 |
| 特征计算服务（实时+离线特征） | P1 | 高 |
| 决策日志与解释（合规要求） | P1 | 中 |
| PMML/ONNX模型推理 | P1 | 中 |
| 灰度决策（策略级5%→100%） | P2 | 中 |
| 决策监控（命中率/延迟/拒绝率） | P2 | 低 |

- 优势：DAG引擎天然适合决策编排，国内无竞品用DAG做实时决策；金融风控/电商推荐付费意愿强；客单价远高于营销自动化
- 风险：需轻量化改造（当前链路太重）；决策与营销用户完全不同需独立销售；对实时性要求极高
- 适合：团队有金融/风控背景

### 方向④：AI原生营销平台（向未来走）

不是在画布里加AI节点，而是让AI成为画布的操盘手。

| 阶段 | 核心能力 | 人月 |
|------|---------|------|
| Phase1(0-6月) AI辅助 | 流失预测、智能时机、内容个性化、AI_LLM节点实化 | 8.0 |
| Phase2(6-18月) AI自动化 | CLV预测、渠道偏好、发送优化、异常检测、自动A/B | 11.5 |
| Phase3(18-36月) AI Agent | NL画布生成、目标驱动编排、跨画布优化Agent、自改进模型 | 18.5 |

- 优势：行业明确趋势，Klaviyo已证明AI预测直接提升ARPU；国内平台AI普遍极弱
- 风险：55.5人月总投入（含14PM前置依赖），周期长；需ML工程人才；AI效果依赖数据量
- 适合：有融资支撑、能承受3年投入周期、目标是行业第一

### 方向⑤：可组合编排平台（泛化走）

DAG引擎不限于营销，任何"事件触发→多步处理→结果输出"的流程都能编排。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| 连接器生态（预置API连接器：飞书/钉钉/S3/数据库） | P0 | 高 |
| 错误恢复与细粒度重试策略 | P0 | 中 |
| 可视化调试器（节点I/O实时查看+断点） | P1 | 中 |
| 执行历史批量回放 | P2 | 中 |
| 子画布并行扇出+结果聚合 | P1 | 中 |
| 表单设计器（非营销场景） | P2 | 高 |
| 流程模板市场（业务场景模板） | P2 | 中 |

- 优势：DAG引擎通用性最强，可服务IT运维/数据管道/业务流程自动化
- 风险：与n8n/Zapier/Temporal直接竞争，它们已成熟且开源；泛化意味着每个垂直场景都不够深
- 适合：不想被限定在营销领域

### 方向⑥：CDP+MA一体化（数据驱动走）

从"用数据做营销"升级为"以数据为核心，营销只是数据的消费场景之一"。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| 数据采集SDK（JS/App端） | P0 | 高 |
| 实时计算管道（Flink/Spark Streaming） | P0 | 极高 |
| 身份图（Identity Graph） | P1 | 高 |
| 数据目录（Data Catalog） | P2 | 中 |
| 数据质量校验 | P1 | 中 |
| 数据仓库直连（Snowflake/BigQuery/Doris） | P1 | 中 |
| 行为分析套件（事件/漏斗/留存/路径） | P0 | 高 |

- 优势：神策已验证路线可行性（估值50亿+RMB）；数据是护城河——客户迁移成本极高
- 风险：CDP建设是重资产，需数据工程团队；神策有4年先发优势；CDP市场正被"可组合CDP"颠覆
- 适合：团队有大数据背景，愿意做重基础设施

### 方向⑦：合规+渠道双护城河（中国基础设施走）

不追功能广度，在PIPL合规+中国渠道上做到极致，成为中国营销的"合规基础设施"。

| 需要增加的能力 | 优先级 | 复杂度 |
|---------------|--------|--------|
| PIPL合规引擎（明示同意/撤回/用户权利/数据删除） | P0 | 中 |
| 合规门（全局发送链路强制拦截） | P0 | 低(代码已存在) |
| 通信短信息合规频控 | P0 | 低 |
| 退订/黑名单管理 | P0 | 低 |
| 抖音/视频号/小红书渠道集成 | P1 | 中 |
| SM2/SM3/SM4国密算法 | P2 | 中 |
| 信创兼容（OceanBase/TiDB/GaussDB） | P2 | 中 |
| 数据出境安全评估工作流 | P2 | 低 |

- 优势：合规是法律红线，海外平台永远无法满足PIPL；中国渠道API需中国实体；合规+渠道是采购硬门槛
- 风险：合规本身不是卖点而是准入条件；纯合规方向天花板低
- 适合：作为方向②的子策略，不宜独立成方向

## 推荐演进路线：②为主轴，⑦为底座，④为远期

### 为什么选②私域中台？

1. **护城河最深**：DAG级画布+企微原生SCRM，国内无人提供。Convertlab有SCRM但画布弱，神策画布更弱
2. **引擎复用度80%**：57个NodeHandler大部分直接复用，只需新增企微/小程序/裂变相关节点
3. **市场时间窗最宽**：私域需求2020年后持续增长，没有衰退迹象
4. **付费意愿最强**：中国B2C品牌在私域上的投入是营销自动化的3-5倍

### 演进路线

```
Phase 0 (0-3月): 生产止血
  ├── 技术债P0: K8s部署、Redis HA、优雅停机、审计日志、CI/CD
  ├── 合规接入: MarketingPolicyService接入发送链路 (#36)
  ├── PIPL合规: 明示同意+退订+频次合规 (#25/#26/#27)
  └── 产出: 可安全上线的合规营销引擎

Phase 1 (3-9月): 私域筑基
  ├── 企微SCRM模块 (客户管理+群聊运营+欢迎语)
  ├── 小程序触达+事件追踪
  ├── 全局疲劳度策略
  ├── 归因引擎 (末次触达归因+归因窗口)
  ├── 模板市场UI
  ├── Webhook/事件订阅
  └── 产出: 私域运营中台MVP

Phase 2 (9-18月): 运营闭环
  ├── 裂变营销引擎 (邀请有礼+拼团+分享解锁)
  ├── 生命周期阶段+自动化编排
  ├── 优惠券全生命周期
  ├── 积分/会员等级
  ├── 渠道回执追踪 (送达/打开/点击/退订)
  ├── 运营日历+冲突检测
  ├── 画布版本管理+回滚
  └── 产出: 完整私域运营中台

Phase 3 (18-36月): AI加持
  ├── 流失预测模型 (Phase1 AI)
  ├── 智能触达时机
  ├── AI内容个性化
  ├── CLV预测+渠道偏好 (Phase2 AI)
  ├── NL画布生成 (Phase3 AI)
  └── 产出: AI原生私域运营平台
```

### 每个阶段的核心差异化

| 阶段 | 一句话定位 | 核心卖点 | 对标 |
|------|-----------|---------|------|
| Phase 0 | "合规的画布引擎" | PIPL合规+DAG画布+5渠道 | 无(国内无人做合规接入) |
| Phase 1 | "画布级私域中台" | DAG画布+企微SCRM+归因 | Convertlab+画布深度 |
| Phase 2 | "全链路运营中台" | 私域+裂变+会员+回执闭环 | 神策运营+画布深度 |
| Phase 3 | "AI原生私域平台" | AI预测+NL画布+自动优化 | Klaviyo+企微(全球唯一) |

### 为什么不选其他方向？

- **① 营销深度**：画布已L2，继续投入边际收益递减。补齐运营管控层后仍是"中国版Braze"，无护城河
- **③ 实时决策**：需轻量化改造当前引擎（Disruptor+Lane链路太重），架构改造成本大；且决策场景与营销用户完全不同，需独立销售
- **④ AI原生**：55.5人月总投入+3年周期，当前团队规模和融资不支持；AI效果依赖数据量，早期客户数据不够
- **⑤ 可组合编排**：与n8n/Zapier/Temporal直接竞争，它们已成熟且开源；泛化意味着每个垂直场景都不够深
- **⑥ CDP+MA**：重资产（OLAP+SDK+实时计算），需数据工程团队；神策有4年先发优势
- **⑦ 合规+渠道**：合规是准入条件不是卖点，天花板低；但作为②的底座策略是必须的
