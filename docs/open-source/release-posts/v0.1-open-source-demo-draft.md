# Draft: Marketing Canvas v0.1 Open Source Demo

Status: draft, not ready to publish.

Marketing Canvas is being prepared as an open-source marketing automation
platform for teams that want to build customer journeys on a visual canvas,
connect capabilities through governed plugins, and review campaign changes
through templates and DSL artifacts.

The v0.1 story is intentionally narrow: make the first journey easy to inspect,
run locally, and explain. The north star is `Time to First Successful Journey`,
measured from opening the repository to running a sample journey dry-run and
seeing trace output.

## What This Draft Can Claim

- A focused product direction: Journey Canvas, plugins, templates, governance,
  traceability, and AI assistance.
- A local development quickstart using Docker-backed dependencies, Spring Boot,
  and Vite.
- Public template documentation for ten official journey scenarios.
- Local MarketingOps as Code examples for Canvas DSL validation and diff.
- Contract documents for plugin manifests, template packs, Canvas DSL, node
  handlers, demo profile behavior, and AI operators.
- Mock-first guidance for external providers and AI demos.

## What This Draft Must Not Claim Yet

- Stable public backend write APIs for plugins, templates, DSL, CLI, or AI.
- Production runtime plugin marketplace readiness.
- Runtime jar hot-loading.
- A finalized public license until it is confirmed by the coordinator.
- A one-command production deployment path.

## Suggested Announcement Copy

Marketing Canvas is an open-source marketing automation platform for teams that
want a visual and reviewable way to operate customer journeys.

With v0.1, the project is narrowing its public surface around a simple workflow:
start the local stack, inspect a journey template, validate or compare a Canvas
DSL artifact, and follow the execution-governance model through dry-run and
trace-oriented docs.

This release draft highlights the foundation:

- Journey Canvas as the primary product experience.
- Template catalog docs for lifecycle, promotion, lead, AI review, risk, and
  private-domain scenarios.
- Build-time plugin contracts instead of runtime hot-loading.
- Local-only DSL validation and diff workflows while backend write APIs remain
  gated.
- Mock provider guidance so demos do not require real SMS, email, coupon,
  approval, AI, or customer credentials.

The next milestones will continue toward plugin and template SDKs, then
MarketingOps as Code with Canvas DSL, CLI, AI draft/audit workflows, and a
playground-oriented demo path.
